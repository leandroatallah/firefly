# SPEC — 060 HTML Report Generator

Scope: Go-based HTML report toolkit under `scripts/gen/`. Pure-stdlib (`html/template`). Two generator entrypoints (diff, domain-docs) share a `base.html` template and a `render` helper package. Output goes to `output/tmp/` (temporary) or `output/docs/` (permanent).

## 1. Package Layout

```
scripts/gen/
├── base.html                  # shared template (CSS inline + blocks)
├── render/
│   ├── render.go              # Renderer: loads base.html, executes child templates
│   └── render_test.go
├── diff/
│   ├── diff.go                # package diff: data types + RenderDiff
│   ├── diff.tmpl              # {{define "title"}}{{define "content"}}
│   └── diff_test.go
├── domaindocs/
│   ├── docs.go                # package domaindocs: data types + RenderDomainDocs
│   ├── docs.tmpl              # defines title/content/sidebar
│   └── docs_test.go
├── cmd/
│   ├── diff/main.go           # scripts/gen/cmd/diff: reads JSON stdin → writes output/tmp/<name>.html
│   └── domain-docs/main.go    # reads JSON stdin → writes output/docs/<name>.html
```

Module path prefix: `firefly/scripts/gen/...` (matches existing go.mod root; verify before implementing — if `scripts/` is outside module scope, place under `internal/tools/gen/` with same structure). Generators are dev tooling, not engine/kit/game code, so the three-layer rule does not apply.

## 2. Shared Renderer [AC-1, AC-2, AC-6]

```go
package render

type Renderer struct {
    base *template.Template // parsed from base.html with default empty blocks
}

func New(baseHTML []byte) (*Renderer, error)
// Parses base.html. Defines default empty blocks "title", "content", "sidebar".

func (r *Renderer) Render(w io.Writer, childTmpl string, data any) error
// Clones base, parses childTmpl source (overriding blocks), executes "base".
```

post:
- `New(nil)` → error "empty base template".
- `New(invalidHTML)` → error wrapping `template.Parse` error.
- `Render` with child omitting `sidebar` → output contains empty `<aside>` section, no panic.
- `Render` with child defining `title` → `<title>` tag contains provided text.
- CSS is inlined in `<style>` tag inside `base.html` (no `<link rel="stylesheet">`, no CDN URLs).

## 3. base.html Structure [AC-1, AC-2, AC-6]

```html
<!doctype html>
<html><head>
  <meta charset="utf-8">
  <title>{{block "title" .}}Firefly Report{{end}}</title>
  <style>/* inlined CSS: layout grid, sidebar 240px, content max-width 960px, monospace diff blocks */</style>
</head><body>
  <header><h1>Firefly</h1></header>
  <div class="layout">
    <aside>{{block "sidebar" .}}{{end}}</aside>
    <main>{{block "content" .}}{{end}}</main>
  </div>
  <footer>Generated by scripts/gen</footer>
</body></html>
```

Constraints:
- No `<script>` tags.
- No external `href`/`src` (CSS, fonts, JS). Validated by test scanning rendered bytes for `https://`, `http://`, `cdn.`.

## 4. Diff Report [AC-3, AC-5]

```go
package diff

type Hunk struct {
    Header string   // e.g. "@@ -10,5 +10,7 @@"
    Lines  []Line
}
type Line struct {
    Kind string // "add" | "del" | "ctx"
    Text string
}
type FileDiff struct {
    Path  string
    Hunks []Hunk
}
type Report struct {
    Title       string
    Explanation string      // markdown-lite: rendered as <p> blocks split on \n\n
    Files       []FileDiff
}

func Render(w io.Writer, r *render.Renderer, rep Report) error
```

Pseudocode:
```
Render:
  validate rep.Title != ""
  exec child template "diff.tmpl" with rep
  child defines:
    title   → rep.Title
    content → for each File: <h2>{{Path}}</h2>; for each Hunk: <pre class="hunk">...</pre>
              line CSS classes: add→.add, del→.del, ctx→.ctx
              prepend explanation paragraphs
```

post:
- Output contains every `FileDiff.Path` wrapped in an `<h2>`.
- Each `Line.Text` appears HTML-escaped (no raw `<`).
- `Kind="add"` line wrapped in element with class `add`; `del` → `del`; `ctx` → `ctx`.
- Sidebar block is empty in output (diff is a temporary report).
- `cmd/diff` writes to `output/tmp/<slug>.html` (slug = lowercase, spaces→`-`, derived from Title).

## 5. Domain Docs Report [AC-4, AC-5, AC-7]

```go
package domaindocs

type DomainEntry struct {
    Name        string
    Slug        string // filename-safe; used for sidebar anchor + output filename
    Description string
    KeyTypes    []string
}
type Index struct {
    Title   string
    Entries []DomainEntry
}

func RenderIndex(w io.Writer, r *render.Renderer, idx Index) error
func RenderEntry(w io.Writer, r *render.Renderer, idx Index, entry DomainEntry) error
```

Pseudocode:
```
sidebarBlock(idx):
  <nav><ul>
    for e in idx.Entries:
      <li><a href="{{e.Slug}}.html">{{e.Name}}</a></li>

RenderIndex:
  child defines title=idx.Title, sidebar=sidebarBlock(idx), content=<h2>Index</h2><ul>...</ul>
RenderEntry:
  child defines title=entry.Name, sidebar=sidebarBlock(idx),
        content=<h1>{{Name}}</h1><p>{{Description}}</p><h3>Key Types</h3><ul>{{range KeyTypes}}<li><code>...</code></li>{{end}}</ul>
```

post:
- Sidebar `<a href>` values are relative (no leading `/`, no scheme). Validated.
- Output file path: `output/docs/<entry.Slug>.html` for entries, `output/docs/index.html` for index.
- Opening `output/docs/index.html` directly via `file://` and clicking sidebar resolves to sibling `<slug>.html` (verified by test asserting `href` equals `<slug>.html`).

## 6. CLI Entrypoints

```
cmd/diff/main.go:
  read JSON from stdin → diff.Report
  Renderer.New(embed base.html)
  ensure output/tmp/ exists (os.MkdirAll 0o755)
  write output/tmp/<slugified Title>.html

cmd/domain-docs/main.go:
  read JSON from stdin → domaindocs.Index
  ensure output/docs/ exists
  write output/docs/index.html
  for each entry: write output/docs/<slug>.html
```

Each `base.html` is embedded via `//go:embed base.html` in the `render` package (or alongside cmd if simpler).
